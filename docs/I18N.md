# Internationalization

Taskboard includes a lightweight German/English language foundation.

## Current approach

UI strings are grouped in `lib/i18n.ts` and accessed through `useI18n`. The language preference is stored with the other app preferences.

## Scope

The current setup covers the main interface labels and messages. It is intentionally simple and does not add a full translation framework.

## Future improvements

A larger version could add stronger type coverage for translation keys, date/number formatting per locale and a more complete translation workflow.
